# Performance Analysis — ISIC-2024 SLICE-3D (no-external-data, efficiency-first)

All numbers below are **out-of-fold (OOF)** on the frozen patient-grouped folds
(`data/folds.parquet`, 5 folds, SEED=42, 401,059 crops / 393 malignant). Every pAUC
is computed **only** via `src/cv.py` (`cv.pauc_above_tpr` / `cv.oof_pauc`), the official
ISIC-2024 partial-AUC above **80% TPR** (range [0, 0.20]; random ≈ 0.02, perfect = 0.20).
No patient straddles a fold; all numbers are **leak-verified by cv-guardian**.

Generated by `reports/perf/make_perf.py`. Figures at DPI 150, Okabe-Ito colorblind-safe palette.

## Headline numbers (recomputed from the OOF parquets)

| Model | OOF pAUC@80%TPR |
|---|---|
| Tabular GBDT | **0.16890** |
| Best image (ConvNeXtV2-nano @224) | 0.15945 |
| **Stack (rank-avg gbdt+r224)** | **0.17430** |

Stack reconstruction (`rank-avg[gbdt, r224]`) recomputes to **0.17430**, matching the
canonical `stack_oof.parquet` exactly — the combiner is genuinely a 2-way rank average.

**CV → private expectation.** The field saw roughly a **0.013–0.021 public→private drop**
on this task even with clean CV (the shake-up was driven by 393 positives + heavy reliance
on external/synthetic data, which we banned). Our headline is **OOF CV 0.1743 (stack) /
0.1689 (tabular)**; treat any private-test projection as headline **minus ~0.01–0.02**, and
note our no-external-data design is specifically what makes the OOF number a more honest
predictor of generalization than the leaderboard-tuned winners.

Tabular 0.16890, best image 0.15945, stack 0.17430. The shaded band is exactly what the

**Takeaway:** in the scored band the stack sits above both single experts — image and tabular

## Fig 3 — Ablation tables
Files: `fig3a_ablation_tabular.png`, `fig3b_ablation_image.png`, `fig3c_ablation_stack.png`

OOF-derived numbers are recomputed from the parquets; the training-time deltas
(Step1/2/3, EMA 0.999) are taken as given per the run logs.

### 3a. Tabular GBDT progression
| Step | Change | OOF pAUC | Δ |
|---|---|---|---|
| 0 | Broken: `is_unbalance` + AUC early-stop, `best_iter=1` | 0.09941 | — |
| 1 | Fix early-stopping / objective | 0.11826 | +0.01885 |
| 2 | Wide patient-relative ugly-duckling features | 0.14420 | +0.02594 |
| 3 | Greysky-bagged + `pxc_` + CatBoost ensemble | **0.16890** | +0.02470 |

**Takeaway:** the broken config (best_iter=1) was scoring barely above 5× random; the
patient-relative feature block (Step 2) is the single biggest jump (+0.026).

### 3b. Image expert — resolution & regularizer
| Config | OOF pAUC |
|---|---|
| convnextv2_nano @128 | 0.15311 |
| **convnextv2_nano @224 +EMA0.995 +mixup** | **0.15945** |
| ↳ same, EMA 0.999 | ~0.146 (HURT, training-time) |
| convnextv2_tiny @224 | 0.15824 |
| swinv2_tiny @256 | 0.10422 |
| eva02_small @336 | 0.09984 |

**Takeaway:** resolution + light EMA(0.995) + mixup on the **small** nano backbone is the
sweet spot; stronger EMA (0.999) over-smooths and heavy transformers (swin/eva02) collapse
to near-random — classic overfitting at 393 positives.

### 3c. Stack combiners
| Combiner | OOF pAUC | Verdict |
|---|---|---|
| **rank-avg [gbdt, r224]** | **0.17430** | WINNER |
| meta-LGBM stacker | 0.17108 | loses to trivial |
| rank [+ weak images] | 0.16679 | dilutes |
| learned per-lesion gate (MoE) | 0.15007 | decisive loss (< tabular 0.16890) |
| + PCA image embeddings into GBDT | < 0.1743 | hurts |

**Takeaway:** the **trivial 2-way rank-average wins**; every learned/heavier combiner loses,
and the MoE gate even falls *below the tabular expert alone* — exactly the negative result the
project predicted at 393 positives.

> Note: a recomputed rank-avg over {gbdt + 4 image OOFs} scores 0.16917; the logged
> "rank [+ weak images]" run (0.16679) used a different weak-image set and is kept as the
> authoritative logged number. Either way the conclusion holds — adding weak images dilutes.

## Fig 5 — Score separation & calibration (stack)
File: `fig5_calibration_separation.png`

Left: log-count histogram of stack OOF scores, malignant (n=393) vs benign (n=400,666) —
malignant mass concentrates near 1.0 while benign mass concentrates near 0.0, with overlap
in the mid-range that the partial-AUC region penalizes. Right: reliability curve (12 quantile
bins). **Takeaway:** the model separates classes well by **rank**, but the raw scores are far
from probability-calibrated (observed malignant fraction stays ≈0 even in the top bin) — as
expected under 0.1% prevalence and a rank-based metric; calibration is not needed for pAUC, so
this is a feature, not a defect.

## Fig 6 — Negative results summary
File: `fig6_negative_results.png`

| Idea tried | Its pAUC | Beaten by | Honest finding |
|---|---|---|---|
| Learned per-lesion gate (MoE) | 0.15007 | 0.16890 (tabular) | **loses to tabular alone** |
| Meta-LGBM stacker | 0.17108 | 0.17430 (rank-avg) | loses to trivial rank-avg |
| + PCA image embeddings into GBDT | < 0.1743 | 0.17430 | embeddings **hurt** |
| Heavy backbones (swinv2 / eva02) | 0.104 / 0.100 | 0.15945 (nano@224) | dominated — overfit @393 pos |
| EMA 0.999 (over-smoothing) | ~0.146 | 0.15945 (EMA0.995) | stronger EMA hurts |

**Takeaway:** every step up in model complexity *lost*. At 393 positives the efficient,
trivial choices (small backbone, rank-average combiner, intrinsic tabular features) are not
just cheaper — they are also more accurate.
