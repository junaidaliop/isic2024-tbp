The problem

  • ISIC-2024 SLICE-3D: detect malignant skin lesions in 3D total-body-photography crops.
  • 401,059 crops · 1,042 patients · 393 malignant / 400,666 benign → a 1,021:1 imbalance.
  • Metric: official pAUC above 80% TPR, range [0, 0.20] (random ≈ 0.02, perfect = 0.20).
  • The whole problem is the imbalance: 393 positives is not much for a deep net alone.

What we deliberately ban

  • No external data — ISIC-2024 SLICE-3D only (no ISIC-2019/2020, no PAD-UFES, no external dermoscopy).
  • No generative / synthetic positives — fabricating pathology breaks label validity.
  • The winners’ ~0.173 used both; matching it is out of reach by construction.

Important

That impossibility is the experiment. So we change the question.

The question we actually answer

What is the best pAUC achievable per unit of inference cost — on SLICE-3D data alone?

  • Not one private-LB number → a quality-vs-cost Pareto frontier.
  • Axes: pAUC vs params · FLOPs · CPU latency.
  • A model only counts if it earns its cost.

CV discipline — the whole game

  • Leaky CV sent teams ~200 places down on the private split.
  • Patient-grouped, target-stratified folds (5, SEED 42), frozen once into data/folds.parquet.
  • No patient straddles folds. One metric, one implementation (src/cv.py), == official scorer.
  • cv-guardian agent holds veto power over splits & metric.

The pipeline

Where the signal lives

  • The dominant signal is the ugly-duckling sign: how anomalous a lesion is vs the same patient’s moles.
  • Patient-relative features (pdev_ / prank_ / pxc_) = ~65% of GBDT gain.
  • Top features: tbp_lv_H (hue, univariate AUC 0.81), pdev_tbp_lv_H, pdev_clin_size.

Every complex idea lost

  • Learned MoE gate 0.150 — below tabular alone.
  • Meta-LGBM stacker 0.171 < trivial 0.174. Image embeddings: hurt. Heavy transformers: collapse.

Why simpler won

  • Every learned add-on must fit parameters from the same 393 positives the experts already used.
  • A rank-average has zero parameters → cannot overfit the folds.
  • At extreme imbalance: bias toward zero-parameter fusion + small backbones — also the cheapest.

Conclusion

  • Defensible claim: SOTA among single-dataset, no-external-data, no-synthetic solutions, on a quality–efficiency frontier.
  • Best constrained point: OOF pAUC 0.17376 @ 15.8 M / 2.46 GFLOPs / 61 ms CPU.
  • Validation discipline + an explicit cost axis give the result teeth.
  • Reproducible: SEED=42, config-driven, make folds → gbdt → vision → frontier → site.
